Câu 2. DNS có vai trò gì trong việc truy cập website?

DNS (Domain Name System) đóng vai trò như một "cuốn danh bạ" của Internet:

  1. Bạn gõ https://google.com vào trình duyệt.
  2. Trình duyệt gửi truy vấn tới DNS Resolver.
  3. DNS Resolver trả về địa chỉ IP tương ứng (ví dụ 142.250.x.x).
  4. Trình duyệt dùng IP đó để kết nối tới máy chủ web và tải nội dung.

Không có DNS, người dùng phải nhớ địa chỉ IP thay vì tên miền dễ nhớ.

Câu 4. Ngrok giải quyết vấn đề gì?

Vấn đề: Máy tính cá nhân nằm sau NAT/Firewall, không có Public IP cố định → người ngoài Internet không thể truy cập localhost của bạn.

Ngrok giải quyết bằng cách:

  1. Tạo một tunnel bảo mật từ máy bạn đến server Ngrok trên Internet.
  2. Cấp cho bạn một URL công khai (ví dụ https://abc123.ngrok.io).
  3. Mọi request tới URL đó được Ngrok chuyển tiếp về localhost.

Ứng dụng: Demo app local, test webhook (GitHub, Stripe…), phát triển app mobile cần HTTPS.

Câu 5. Nginx với vai trò Reverse Proxy hoạt động ra sao?

Nginx đứng trước các server ứng dụng, nhận request từ client rồi chuyển tiếp về backend:

Client (Browser)
      │  request https://example.com
      ▼
  [ Nginx – Reverse Proxy ]
      │  forward tới backend
      ▼
  [ App Server: Node.js / Python... ]
      │  response
      ▼
  [ Nginx ] ──► trả về Client

Lợi ích:

  • Load Balancing Phân phối traffic cho nhiều backend.
  • SSL Termination Nginx xử lý HTTPS, backend chỉ dùng HTTP nội bộ.
  • Caching Giảm tải cho backend.
  • Bảo mật Che giấu địa chỉ thực của server ứng dụng.
  • Gzip Nén dữ liệu trước khi gửi về client.

Câu 6. Tại sao website ngày nay cần HTTPS thay vì HTTP?

  1. Bảo mật dữ liệu: HTTPS mã hóa toàn bộ dữ liệu (mật khẩu, thẻ ngân hàng…), chống nghe lén và tấn công Man-in-the-Middle.
  2. Xác thực danh tính: Chứng chỉ TLS/SSL xác nhận đây đúng là website bạn muốn truy cập, chống phishing.
  3. SEO và uy tín: Google ưu tiên xếp hạng trang HTTPS; trình duyệt cảnh báo "Not Secure" với HTTP.
  4. HTTP/2 và hiệu năng: Hầu hết trình duyệt chỉ hỗ trợ HTTP/2 (nhanh hơn) qua HTTPS.
  5. API hiện đại: Geolocation, Service Worker, camera… chỉ hoạt động trên HTTPS.
  6. Niềm tin người dùng: Ổ khóa xanh 🔒 tạo sự tin tưởng khi người dùng nhập thông tin cá nhân.
